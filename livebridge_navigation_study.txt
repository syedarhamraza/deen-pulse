LIVEBRIDGE SWIPE NAVIGATION STUDY
===================================

This document details the architectural approach used by the Flutter-based LiveBridge project
(https://github.com/appsfolder/livebridge) to implement gesture-driven, swipable page navigation
synchronized in real-time with a floating bottom navigation bar, and details how you can
replicate this in React Native.

1. FLUTTER ARCHITECTURE IN LIVEBRIDGE
======================================

The navigation system of LiveBridge consists of three core components:
a) A Parent Screen Container (StatefulWidget)
b) A PageView Widget (Gesture Swiper)
c) A Custom Floating Navigation Bar (Canvas-interpolated Layout)

A. The Page Controller and Progress Calculation
------------------------------------------------
In `lib/screens/redesign/home_redesign_screen.dart`, LiveBridge defines:
* `late final PageController _pageController;` initialized in `initState()` with the current tab index.
* A real-time helper getter `_pageProgress`:
  ```dart
  double get _pageProgress {
    if (!_pageController.hasClients) {
      return _currentTabIndex.toDouble();
    }
    return _pageController.page ?? _currentTabIndex.toDouble();
  }
  ```
  - Unlike standard index listeners which only fire when a scroll fully snaps, `_pageController.page`
    returns a continuous fractional value (e.g. `0.23` when swiping from tab 0 to tab 1).
  - This fractional value is passed directly to the floating bottom navigation bar.

B. Tab Selection & Drag Routines
---------------------------------
To handle user input cleanly:
* Programmatic Navigation (Tapping a tab):
  ```dart
  Future<void> _handleTabSelected(int index) async {
    // Determine scroll duration based on distance to the tab
    await _pageController.animateToPage(
      index,
      duration: Duration(milliseconds: 320 + ((index - baseIndex).abs() * 120)),
      curve: Curves.easeOutCubic,
    );
  }
  ```
* Gesture Navigation (Swiping screens):
  - The standard `PageView(controller: _pageController, onPageChanged: (index) { ... })` callback
    handles updating the active index state when scroll momentum finishes.

C. Fluid Canvas Rendering in LbBottomFloatingNav
-------------------------------------------------
In `lib/widgets/redesign/lb_bottom_floating_nav.dart`, the floating bottom tab bar draws its
highlight bubble. It takes `selectionProgress` (which is `_pageProgress` from above).
* Transition Scale Math:
  As the screen is dragged, the bubble changes shape (stretches and squeezes) using interpolation:
  ```dart
  double _pageSwipeScale(double progress) {
    final double offsetFromTab = (progress - progress.roundToDouble()).abs();
    if (offsetFromTab < 0.001) return 1.0;
    
    final double normalized = (offsetFromTab / 0.5).clamp(0.0, 1.0);
    return lerpDouble(
      1.0,
      LbMotion.bottomNavHighlightSwipeScale,
      Curves.easeOut.transform(normalized),
    );
  }
  ```
* Horizontal translation (`translateX`) maps directly to:
  `indicatorOffset = (selectionProgress * tabWidth)`
  This allows the bubble to glide smoothly, matching the user's thumb exactly.
